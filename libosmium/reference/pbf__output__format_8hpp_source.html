38 About the .osm.pbf file format
39 This is an excerpt of <http://wiki.openstreetmap.org/wiki/PBF_Format>
40 
41 The .osm.pbf format and it's derived formats (.osh.pbf and .osc.pbf) are encoded
42 using googles protobuf library for the low-level storage. They are constructed
43 by nesting data on two levels:
44 
45 On the lower level the file is constructed using BlobHeaders and Blobs. A .osm.pbf
46 file contains multiple sequences of
47  1. a 4-byte header size, stored in network-byte-order
48  2. a BlobHeader of exactly this size
49  3. a Blob
50 
51 The BlobHeader tells the reader about the type and size of the following Blob. The
52 Blob can contain data in raw or zlib-compressed form. After uncompressing the blob
53 it is treated differently depending on the type specified in the BlobHeader.
54 
55 The contents of the Blob belongs to the higher level. It contains either an HeaderBlock
56 (type="OSMHeader") or an PrimitiveBlock (type="OSMData"). The file needs to have
57 at least one HeaderBlock before the first PrimitiveBlock.
58 
59 The HeaderBlock contains meta-information like the writing program or a bbox. It may
60 also contain multiple "required features" that describe what kinds of input a
61 reading program needs to handle in order to fully understand the files' contents.
62 
63 The PrimitiveBlock can store multiple types of objects (i.e. 5 nodes, 2 ways and
64 1 relation). It contains one or more PrimitiveGroup which in turn contain multiple
65 nodes, ways or relations. A PrimitiveGroup should only contain one kind of object.
66 
67 There's a special kind of "object type" called dense-nodes. It is used to store nodes
68 in a very dense format, avoiding message overheads and using delta-encoding for nearly
69 all ids.
70 
71 All Strings are stored as indexes to rows in a StringTable. The StringTable contains
72 one row for each used string, so strings that are used multiple times need to be
73 stored only once. The StringTable is sorted by usage-count, so the most often used
74 string is stored at index 1.
75 
76 A simple outline of a .osm.pbf file could look like this:
77 
78  4-bytes header size
79  BlobHeader
80  Blob
81  HeaderBlock
82  4-bytes header size
83  BlobHeader
84  Blob
85  PrimitiveBlock
86  StringTable
87  PrimitiveGroup
88  5 nodes
89  PrimitiveGroup
90  2 ways
91  PrimitiveGroup
92  1 relation
93 
94 More complete outlines of real .osm.pbf files can be created using the osmpbf-outline tool:
95  <https://github.com/MaZderMind/OSM-binary/tree/osmpbf-outline>
96 */

149  std::string serialize_blob(const std::string& type, const google::protobuf::MessageLite& msg, bool use_compression) {
150  OSMPBF::Blob pbf_blob;
151 
152  {
153  std::string content;
154  msg.SerializeToString(&content);
155 
156  pbf_blob.set_raw_size(content.size());
157 
158  if (use_compression) {
159  pbf_blob.set_zlib_data(osmium::io::detail::zlib_compress(content));
160  } else {
161  pbf_blob.set_raw(content);
162  }
163  }
164 
165  std::string blob_data;
166  pbf_blob.SerializeToString(&blob_data);
167 
168  OSMPBF::BlobHeader pbf_blob_header;
169  pbf_blob_header.set_type(type);
170  pbf_blob_header.set_datasize(blob_data.size());
171 
172  std::string blob_header_data;
173  pbf_blob_header.SerializeToString(&blob_header_data);
174 
175  uint32_t sz = htonl(blob_header_data.size());
176 
177  // write to output: the 4-byte BlobHeader-Size followed by the BlobHeader followed by the Blob
178  std::string output;
179  output.reserve(sizeof(sz) + blob_header_data.size() + blob_data.size());
180  output.append(reinterpret_cast<const char*>(&sz), sizeof(sz));
181  output.append(blob_header_data);
182  output.append(blob_data);
183 
184  return output;
185  }

349  void map_string_ids() {
350  // test, if the node-block has been allocated
351  if (pbf_nodes) {
352  // iterate over all nodes, passing them to the map_common_string_ids function
353  for (int i=0, l=pbf_nodes->nodes_size(); i<l; i++) {
354  map_common_string_ids(pbf_nodes->mutable_nodes(i));
355  }
356 
357  // test, if the node-block has a densenodes structure
358  if (pbf_nodes->has_dense()) {
359  // get a pointer to the densenodes structure
360  OSMPBF::DenseNodes* dense = pbf_nodes->mutable_dense();
361 
362  // in the densenodes structure keys and vals are encoded in an intermixed
363  // array, individual nodes are seperated by a value of 0 (0 in the StringTable
364  // is always unused). String-ids of 0 are thus kept alone.
365  for (int i=0, l=dense->keys_vals_size(); i<l; i++) {
366  // map interim string-ids > 0 to real string ids
367  uint16_t sid = dense->keys_vals(i);
368  if (sid > 0) {
369  dense->set_keys_vals(i, string_table.map_string_id(sid));
370  }
371  }
372 
373  // test if the densenodes block has meta infos
374  if (dense->has_denseinfo()) {
375  // get a pointer to the denseinfo structure
376  OSMPBF::DenseInfo* denseinfo = dense->mutable_denseinfo();
377 
378  // iterate over all username string-ids
379  for (int i=0, l= denseinfo->user_sid_size(); i<l; i++) {
380  // map interim string-ids > 0 to real string ids
381  uint16_t user_sid = string_table.map_string_id(denseinfo->user_sid(i));
382 
383  // delta encode the string-id
384  denseinfo->set_user_sid(i, m_delta_user_sid.update(user_sid));
385  }
386  }
387  }
388  }
389 
390  // test, if the ways-block has been allocated
391  if (pbf_ways) {
392  // iterate over all ways, passing them to the map_common_string_ids function
393  for (int i=0, l=pbf_ways->ways_size(); i<l; i++) {
394  map_common_string_ids(pbf_ways->mutable_ways(i));
395  }
396  }
397 
398  // test, if the relations-block has been allocated
399  if (pbf_relations) {
400  // iterate over all relations
401  for (int i=0, l=pbf_relations->relations_size(); i<l; i++) {
402  // get a pointer to the relation
403  OSMPBF::Relation* relation = pbf_relations->mutable_relations(i);
404 
405  // pass them to the map_common_string_ids function
406  map_common_string_ids(relation);
407 
408  // iterate over all relation members, mapping the interim string-ids
409  // of the role to real string ids
410  for (int mi=0, ml=relation->roles_sid_size(); mi<ml; mi++) {
411  relation->set_roles_sid(mi, string_table.map_string_id(relation->roles_sid(mi)));
412  }
413  }
414  }

424  void map_common_string_ids(TPBFObject* in) {
425  // if the object has meta-info attached

607  void write_dense_node(const osmium::Node& node) {
608  // add a DenseNodes-Section to the PrimitiveGroup
609  OSMPBF::DenseNodes* dense = pbf_nodes->mutable_dense();
610 
611  // copy the id, delta encoded
612  dense->add_id(m_delta_id.update(node.id()));
613 
614  // copy the longitude, delta encoded
615  dense->add_lon(m_delta_lon.update(lonlat2int(node.lon())));
616 
617  // copy the latitude, delta encoded
618  dense->add_lat(m_delta_lat.update(lonlat2int(node.lat())));
619 
620  // in the densenodes structure keys and vals are encoded in an intermixed
621  // array, individual nodes are seperated by a value of 0 (0 in the StringTable
622  // is always unused)
623  // so for three nodes the keys_vals array may look like this: 3 5 2 1 0 0 8 5
624  // the first node has two tags (3=>5 and 2=>1), the second node does not
625  // have any tags and the third node has a single tag (8=>5)
626  for (const auto& tag : node.tags()) {
627  dense->add_keys_vals(string_table.record_string(tag.key()));
628  dense->add_keys_vals(string_table.record_string(tag.value()));
629  }
630  dense->add_keys_vals(0);
631 
632  if (m_should_add_metadata) {
633  // add a DenseInfo-Section to the PrimitiveGroup
634  OSMPBF::DenseInfo* denseinfo = dense->mutable_denseinfo();
635 
636  denseinfo->add_version(node.version());
637 
638  if (m_add_visible) {
639  denseinfo->add_visible(node.visible());
640  }
641 
642  // copy the timestamp, delta encoded
643  denseinfo->add_timestamp(m_delta_timestamp.update(timestamp2int(node.timestamp())));
644 
645  // copy the changeset, delta encoded
646  denseinfo->add_changeset(m_delta_changeset.update(node.changeset()));
647 
648  // copy the user id, delta encoded
649  denseinfo->add_uid(m_delta_uid.update(node.uid()));
650 
651  // record the user-name to the interim stringtable and copy the
652  // interim string-id to the pbf-object
653  denseinfo->add_user_sid(string_table.record_string(node.user()));
654  }
